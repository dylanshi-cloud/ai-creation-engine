# 路线图 · 三阶段执行计划

> 从一条窄路走通，再横向扩展
> 更新时间: 2026年6月

---

## 战略原则

1. **先窄后宽**：先在一个物理域跑通端到端，再横向迁移到更多域。不要试图一开始就覆盖所有制造业。
2. **仿真优先**：用大规模仿真数据驱动模型进化，物理验证只做最终确认。仿真是物理AI的"互联网文本"。
3. **Layer 0 思维**：用现有AI工具建造这个引擎本身。不等待技术突破——利用已经存在的工具。

---

## Phase 1：验证期（0-3个月）

**目标：** 选一个域，跑通"AI设计→FEA验证→AI迭代"的完整循环。

### 选域：注塑件支架

理由：
- 结构相对简单（壁厚、加强筋、圆角、孔位）
- FEA成熟（模态+结构耦合）
- 打样便宜，迭代快（义乌供应链）
- 有实际经济价值

### 里程碑

| # | 时间 | 完成标准 |
|:---|:---|:---|
| M1 | 第2周 | 环境就绪：CadQuery+CalculiX安装、第一个参数化脚本跑通 |
| M2 | 第4周 | FEA验证跑通：一个设计走完CAD→FEA→读结果→修改→再FEA循环 |
| M3 | 第8周 | 自动化管线：AI可以自主完成3轮以上的设计→FEA→修改循环 |
| M4 | 第12周 | 打样验证：最优设计成功打样并通过物理测试，累计100+配对数 |

### 工具链

```
Claude (AI编排) → CadQuery (参数化CAD脚本) → STEP文件
→ CalculiX (FEA) → 结果数据 (最大应力/形变/质量)
→ Claude 分析结果 → 修改参数 → 重新生成 → 直到收敛
```

## Phase 2：数据引擎期（3-6个月）

**目标：** 数据规模化 + 训练物理直觉模型，减少FEA调用量一个数量级。

### 里程碑

| # | 时间 | 完成标准 |
|:---|:---|:---|
| M5 | 第16周 | 数据规模化：1,000+个设计-仿真配对数据入库 |
| M6 | 第20周 | 直觉模型：预测通过/不通过准确率>85%，应力预测MAE<±15% |
| M7 | 第24周 | 调度器集成：生成器→直觉筛选→FEA确认的管线跑通 |

### 关键交付物

- 1,000+条 (几何参数→FEA结果) 配对数据，公开数据集
- 物理直觉预测模型（轻量级，XGBoost起步）
- 迭代调度器脚本

---

## Phase 3：扩展期（6-12个月）

**目标：** 横向扩展到更多物理域，纵向深化到输出可制造文件。

### 候选扩展域

| 域 | 难度 | 新增要素 |
|:---|:---|:---|
| 钣金件（非标支架） | 中 | 折弯展开、焊接仿真、成本优化 |
| 铝挤型散热片 | 中 | 热仿真耦合、空气对流 |
| 简单装配体（<10零件） | 高 | 接触非线性、装配序列 |
| CNC加工件 | 中高 | 可制造性约束（刀具可达性、夹具） |

每个新域重复 Phase 1→Phase 2 循环。

### 里程碑

| # | 时间 | 完成标准 |
|:---|:---|:---|
| M8 | 第36周 | 3个域跑通端到端管线 |
| M9 | 第48周 | 从需求到制造文件的人工介入时间<30分钟 |
| M10 | 第52周 | 打样一次通过率>70% |

## 实践指南

### 如何参与任何阶段

1. **选择你熟悉的物理域**——你最了解什么零件？什么工艺？
2. **阅读 ** [philosophy.md](philosophy.md) 理解这套方法的核心思想
3. **阅读 ** [architecture.md](architecture.md) 理解四层架构
4. **从Phase 1的步骤开始**——在你的域里跑通第一个设计→FEA循环
5. **分享你的经验**——在issue里记录你的域、工具链、遇到的坑、解决的方法

### 判断是否选对了域

- 该零件的CAD可以参数化描述（不用从零画，用算法生成）
- 该零件的FEA仿真成熟可靠（仿真结果和真实测试偏差已知）
- 你/你的合作伙伴能拿到打样结果（不然永远不知道仿真准不准）
- 这个域有实际价值（优化后能省钱/省材料/省时间）

## 长期愿景

当一个域跑通、数据积累足够后，整套管线可以迁移到下一个域。每个域积累的不仅仅是代码——是(设计参数→物理结果)的配对数据。

**这些数据本身，就是物理AI的"互联网文本"。**

当配对数据量级达到亿级、模型从简单ML进化到物理Transformer的那一天——

物理AI的"Transformer时刻"就到了。
